## 📋 Этапы работы

### 1️⃣ Первичный анализ данных
- Размер датасета (количество строк и столбцов)
- Типы признаков (числовые, категориальные)
- Наличие пропусков
- Распределение классов (баланс/дисбаланс)
- Базовые статистики (describe, info)
- Возможные проблемы в данных

### 2️⃣ Подготовка данных
- Обработка пропусков (если есть)
- Кодирование категориальных признаков
- **Масштабирование признаков** (StandardScaler / MinMaxScaler)
- Разделение на train/test

**Почему масштабирование важно для KNN?**  
KNN основан на расстояниях между объектами. Признаки с большими значениями (например, цена 0-500) будут доминировать над признаками с малыми значениями (возраст 0-80), искажая расстояния.

**Почему нельзя подбирать параметры на тестовой выборке?**  
Это приведёт к утечке данных (data leakage) — модель будет "подглядывать" в ответы, и качество на тесте будет завышено, не отражая реальную работу на новых данных.

### 3️⃣ Обучение KNN
Исследование влияния гиперпараметров:

| Параметр | Значения для экспериментов |
|----------|----------------------------|
| `n_neighbors` | 1, 3, 5, 7, 10, 15, 20 |
| `weights` | 'uniform', 'distance' |
| `metric` | 'euclidean', 'manhattan', 'minkowski' |

### 4️⃣ Подбор гиперпараметров с кросс-валидацией
- Использование **GridSearchCV** для поиска оптимальных параметров
- Сравнение результатов:
  - без масштабирования
  - после масштабирования
  - с разными метриками расстояния
  - с разными значениями k

### 5️⃣ Оценка качества модели
Метрики для оценки:
- **Accuracy** — общая точность
- **Precision** — точность (минимум ложных срабатываний)
- **Recall** — полнота (минимум пропущенных объектов)
- **F1-score** — гармоническое среднее
- **Confusion matrix** — матрица ошибок
